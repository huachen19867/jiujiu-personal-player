# 2026-06-23 技术日志
## 自动读取本地歌单、虚拟滚动与随机模式重构

本轮主题是三个并行功能，对应 versionCode 14 / versionName 1.0.13，改动覆盖 14 个文件、+445/-33 行。

第一个是"自动读取本地"歌单。`LocalMusicPickerPlugin.java` 新增 `scanAudioFiles()` 方法，通过 MediaStore 查询手机媒体库中的音频文件，不再依赖手工选文件。权限处理按 Android 13+ 用 `READ_MEDIA_AUDIO`、旧版本回退到 `READ_EXTERNAL_STORAGE`，通过 Capacitor `@Permission` 注解统一管理请求和回调。前端 `nativeBridge.ts` 增加可选的 `scanAudioFiles` 接口，`App.tsx` 在 activePlaylist 为 `auto-local` 时调用扫描而非弹出文件选择器。

第二个是虚拟滚动。`Playlist.tsx` 设置阈值 120 首，超过后只渲染可视区域 + 上下各 8 行 overscan，用 `paddingTop`/`paddingBottom` 维持滚动条总高度。通过 `resize` 事件监听容器高度变化。实测 4000 首歌时 DOM 节点控制在 80 个以内，避免 WebView 黑屏。`.is-virtualized` 样式类加了 `max-height: min(620px, 68vh)`，桌面断点用 `:not(.is-virtualized)` 豁免。

第三个是随机模式重做。`useMusicPlayer.ts` 用 `shuffledPlaybackQueueRef` 缓存完整打乱队列，进入 shuffle 时 `createAndStoreShuffleQueue` 一次性生成并用 Fisher-Yates 洗牌。`getEffectivePlaybackQueueFromRefs` 在 shuffle 模式复用已有队列（通过 `hasSameQueueSongs` 比较 song id 集合），歌曲集合变化时才重新打乱。方向计算在 shuffle 时当作 repeat-all 处理。`cycleMode()` 进入 shuffle 时可选以当前歌曲开头，退出时清空缓存队列。`togglePlay` 在无当前歌曲时从跨歌单队列第一首开始播，导出 `canPlay` 控制播放按钮 disabled 状态。

版本号自增从 1.0.10 → 1.0.13（versionCode 11 → 14），README、package.json 的 copy-apk 路径同步更新。测试覆盖扫描入口、虚拟滚动渲染数量、跨视图空歌单播放启动、重命名歌单改用 find 定位。验证结果：`npm test -- --run` 6 个测试文件、56 条用例全通过；`npm run build` TypeScript + Vite 生产构建通过。
