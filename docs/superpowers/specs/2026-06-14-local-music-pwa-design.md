# 本地音乐播放器 PWA 设计说明

日期：2026-06-14

## 目标

构建一个手机端本地音乐播放器，让用户不依赖网易云等在线音乐 App，也能在手机浏览器中选择本地下载的音乐并播放。首版聚焦本地导入和稳定播放，不做账号、云同步、歌词库、在线搜索和推荐。

## 产品边界

首版入口就是播放器本体，不做营销落地页。用户进入后可以通过“选歌”手动多选音频文件，也可以通过“导入文件夹”尝试批量读取目录下的音频文件。应用只使用浏览器本地能力读取文件，文件不会上传到服务器。

目录导入采用能力增强策略：支持 `showDirectoryPicker` 的浏览器使用目录选择器递归扫描音频文件；不支持时提示用户改用多选文件。由于 iOS Safari、部分 Android WebView 对目录权限支持不稳定，文件多选是首版稳定兜底能力。

## 体验设计

界面按手机端触控优先设计：上方展示当前歌曲信息和播放状态，中间是进度条和主播放控制，下方是可滚动播放列表。导入操作放在固定工具区，避免用户进入空列表后不知道下一步。视觉方向选择“克制的黑胶设备感”：深色底、清晰层级、少量高对比色用于播放状态，不使用泛滥的紫蓝渐变。

歌曲条目显示文件名、格式和时长；无法读取时长时显示占位状态。当前播放歌曲高亮。列表中支持单曲移除；清空列表需要明确按钮触发。

## 功能设计

首版功能包括本地导入、播放/暂停、上一首、下一首、进度拖动、音量、顺序播放、单曲循环、列表循环、随机播放、当前播放高亮、移除歌曲和清空列表。

播放列表元数据保存到 `localStorage`，用于保留上次的歌名、顺序和模式偏好。音频文件对象 URL 只在当前会话内创建，页面刷新后需要用户重新选择文件授权。这是为了避开移动浏览器对持久文件句柄、存储额度和隐私权限的兼容风险。

## 架构

使用 Vite + React + TypeScript 构建 PWA。核心播放逻辑放在自定义 hook 中，UI 组件只关心展示和用户操作。文件导入逻辑独立成工具模块，负责过滤音频、递归目录、生成歌曲模型。浏览器音频播放使用原生 `HTMLAudioElement`，通过 React 状态同步播放进度、播放模式和列表状态。

## 数据模型

歌曲模型包含 `id`、`name`、`artist`、`album`、`type`、`size`、`duration`、`file`、`url`。其中 `file` 和 `url` 仅当前会话有效；保存到本地时只保留可序列化元数据。播放模式使用 `sequence`、`repeat-one`、`repeat-all`、`shuffle`。

## 错误处理

如果用户取消选择文件，界面保持当前状态并不报错。如果导入的文件夹或多选结果中没有音频文件，显示轻提示。播放失败时跳过自动播放假设，保留当前歌曲并提示文件可能无法播放。切换或移除歌曲时释放不用的 object URL，避免长时间使用造成内存堆积。

## 测试策略

单元测试覆盖音频文件过滤、歌曲模型生成、播放模式下一首/上一首选择逻辑和播放列表持久化。组件测试覆盖空列表导入入口、歌曲列表渲染、播放按钮状态、模式切换和移除歌曲。端到端/浏览器验证覆盖手机宽度下页面可见、控件不重叠、本地文件选择入口可用、PWA 构建成功。

## 非目标

首版不做在线曲库、登录账号、云同步、歌词解析、专辑封面自动识别、后台锁屏控制、系统级媒体库扫描和真正安装包打包。Android APK 可在 PWA 主线稳定后用 Capacitor 另开阶段处理。
